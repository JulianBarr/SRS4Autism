# 📂 【架构蓝图】CUMA 认知模块：面孔隐式特征提取 (Facial Recognition)

**归属目录**:: 30_Cognition
**核心目标**:: 破解 ASD 儿童的“面孔失认症（局部处理）”，通过算法逆向工程，提炼面部核心信息熵，结合 FSRS 烧录全局认知体系。

## 🧠 一、 临床本体论 (The Ontology)
人类认脸不需要看清每个毛孔，只需提取约 10 个 Bit 的高熵特征：
- **绝对锚点（权重 50%）**：眉眼组合 (Eye-Eyebrow Region)。眉毛的弧度/粗细，是识别身份的第一特征。
- **空间基准（权重 30%）**：瞳距 (IPD - Interpupillary Distance)。人脸的几何测算必须以双眼瞳孔间距为基准分母，计算其他五官的相对比例（二阶空间结构）。
- **动态干扰（需隔离）**：嘴部与下半脸。儿童易受嘴部动态和声音吸引，从而错过眼部的核心身份信息。

## 🔨 二、 CUMA “三板斧” 战术 (The Strategy)
1. **Ontology（定义）**：确定要练的脸部特征（如：眼距、眉高）。
2. **Reverse-engineer（逆向提取）**：用代码将照片中的冗余信息（皮肤、光影）物理打码或剥离，强制暴露核心特征。
3. **SRS / FSRS（烧录）**：将提取出的靶向图片做成 Anki 牌组，通过海量间隔重复，把“认脸特征”刻入肌肉记忆。

## 🃏 三、 牌组自动化生成流水线 (Deck Pipeline)
利用代码批量处理人脸照，按阶梯生成 FSRS 训练卡片：
- **Stage 1: 极简隔离（只认眉眼）**。提取特征点生成线稿，提问：“这是谁的眼睛？”
- **Stage 2: 物理打码（强制注视）**。保留真实眉眼，其余面部区域（特别是嘴部）进行重度高斯模糊。
- **Stage 3: 空间扭曲（训练全局比例）**。利用 OpenCV 篡改图像，拉宽眼距或拉长人中，提问：“哪张脸看起来很奇怪？”

## 💻 四、 技术栈与开源外挂 (Tech Stack)
- **核心视觉引擎**：MediaPipe Face Mesh (提取 468 个三维面部节点)。
- **图像处理**：OpenCV (负责裁剪、蒙版 Mask、高斯模糊)。
- **牌组编译**：Python `genanki` 库 (一键生成 `.apkg`)。
